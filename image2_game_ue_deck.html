小例子

轻一下，还是隆重一下？

同一个保存反馈，先看两种假设，再判断值不值得打断玩家。

方案 A｜非打断式状态反馈

非打断式状态反馈：玩家保持操作流，提示只完成“确认已保存”的任务。
  • 高频操作，玩家还要继续编辑。
  • 飘字 / Toast / 轻微发光，短暂停留后自动消失。
核心判断：确认即可，不打扰。

方案 B｜打断式演出反馈

打断式演出反馈：把保存动作包装成完成时刻，适合首次解锁、稀有配方或关键节点。
  • 首次保存重要配方，或解锁稀有配方。
  • 书本飞入、翻页记录、成功面板，主动制造仪式感。
核心判断：把保存变成一个小成就。

[交互约束] UE 的底线

以输入 UE 结构图为布局和信息层级参考，生成高保真暗黑中世纪幻想游戏 UI。保持原图主要模块位置、文字内容和交互关系，不重新设计功能结构。

[风格定义] 视觉基调

美术风格为古旧魔法食谱书与地下酒馆氛围，整体厚重、复古、神秘、成熟。使用旧羊皮纸、深棕皮革、古书页、暗金雕花边框、暗红按钮、金属浮雕、烛光阴影等材质。背景为虚化的中世纪厨房或地下大厅，石墙、木梁、暖色火光和暗部环境光营造沉浸感。

[材质细节] 界面主体

UI 主体像一本打开的古老食谱书，纸张泛黄、有污渍、折痕、磨损边缘和层叠阴影；左侧面板偏深色皮革与旧纸质感，中间书页突出当前配方，右侧列表卡片带羊皮纸纹理和红色选中描边。料理图标和食材图标需要精致、半写实、有体积感和幻想游戏质感。

[光影与可读性] 上线品质

色彩以黑棕、深咖、旧纸黄、暗金、暗红为主，绿色数值、红色强调框和紫色图标光晕作为点缀。按钮具有暗红宝石/金属质感，带金色描边、浮雕、高光和可点击反馈。整体光影真实，前景 UI 清晰，背景虚化，有暗角和电影感。中文文字清晰可读，最终效果像成熟上线品质的 RPG 烹饪系统界面。

game-ui-concept-board Skill 完整 Skill：标准流程与图像生成规则
标准流程：把 IP 气质、字体、配色、设计语言和综合方案拆成可复用步骤，输出可横向比较、可挑选、可混搭、可继续深化的 UI 方向方案板。
阶段 1：世界观 / IP 理解

提炼：

  • 世界观气质
  • 时代背景
  • 力量体系
  • 情绪基调
  • 审美来源

输出：3–6 套核心意向方向图

阶段 2：字体方案

每套方案必须包含：

  • 大标题字体
  • 副标题字体
  • 正文字体
  • 数字字体

规则：不同方案的字体必须是结构差异，不是换颜色

输出：字体方案大图

阶段 3：配色方案

默认输出：6 套配色方案

  • 主色、辅助色、强调色、中性色
  • 色彩比例
  • 应用场景
  • UI 小样

规则：配色服务 UI 信息层级；场景图不应被 UI 色系染色；模板背景保持中性，便于横向对比。

输出：配色方案大图

阶段 4：设计语言方案

每套包含：构图语言、材质语言、组件语言、装饰语言

必须展示：

  • 按钮
  • 弹窗
  • 卡片
  • 面板
  • 图标 / 装饰元素

输出：设计语言方案大图

阶段 5：综合方案

默认输出：3 套完整方向

  • 核心意向
  • 字体系统
  • 配色系统
  • 设计语言
  • UI 示例

输出：6000×4000 大图方案板

图像生成规则
  • 默认大图尺寸：6000×4000
  • 黑色或中性背景用于方案板，不被单个方案色系影响
  • 每套方案之间必须有明显差异：字体差异、配色差异、组件形态差异、设计语言差异
  • 避免“同一模板换色”
  • 避免“场景图被 UI 色系染色”
  • 输出应便于用户挑选、混搭、继续深化
